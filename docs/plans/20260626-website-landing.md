# План — сайт Pudding (лендинг + Privacy + Terms)

## Overview

Сделать статический сайт бренда **Pudding**: продающий лендинг + страницы Privacy
Policy и Terms of Service (EULA), двуязычно **RU + EN**. Сайт закрывает не только
маркетинг, но и **обязательные требования Apple**: Privacy Policy URL (Guideline
5.1.1) и Support/contact URL (1.5) — то, что нужно для подачи в App Store. Бонус:
privacy-стойкость приложения («данные не покидают телефон») здесь — продающий
аргумент, а не только compliance.

Стек: **чистая статика HTML/CSS** (без фреймворков и сборки), деплой на
**Vercel/Netlify** из git. Бренд — палитра и маскот из приложения (`Theme.swift`),
чтобы сайт и апп смотрелись одним продуктом.

**Вне скоупа (и почему):**
- **Бэкенд / формы / CMS / блог** — статике это не нужно; контакт через `mailto:`.
- **Аналитика/пиксели** — осознанно нет: «без слежки» — часть позиционирования (и
  согласуется с «Data Not Collected» приложения).
- **Покупка домена** — 🙋 USER (по ресёрчу `getpudding.com` свободен; до этого работаем
  на превью-URL Vercel/Netlify).
- **Юр-вычитка под GDPR/COPPA по рынкам** — нужен юрист; здесь готовим честный
  compliance-текст под требования Apple Review, не юр-документ.
- **Реальные скриншоты/видео аппа** — ставим плейсхолдеры, заменим экспортом из симулятора (🙋 USER).

## Context

- **Files involved (всё новое, каталог `site/`):**
  - Create: `site/index.html` (RU лендинг), `site/en/index.html` (EN лендинг)
  - Create: `site/privacy.html`, `site/en/privacy.html` — Privacy Policy
  - Create: `site/terms.html`, `site/en/terms.html` — Terms of Service / EULA
  - Create: `site/styles.css` — общие стили (палитра + типографика бренда)
  - Create: `site/assets/pudding.svg` — маскот в виде inline/SVG (повтор форм из `Mascot.swift`)
  - Create: `site/assets/` — плейсхолдеры скриншотов, App Store badge, og-image
  - Create: `site/robots.txt`, `site/site.webmanifest`, favicon
  - Create: `vercel.json` **или** `netlify.toml` — конфиг статического деплоя + заголовки
  - Modify (связка с приложением): `Sources/App/AppLinks.swift` — `privacyPolicyURL`,
    `termsURL`, `supportEmail`, `methodologyInfoURL` указывают на страницы сайта
  - Modify: `docs/SUBMISSION.md` — вписать боевые URL после деплоя
- **Бренд (из `Theme.swift`, чтобы 1-в-1 с аппом):** акцент `#FC7D4F` → `#F75470`
  (коралл→малина градиент), `#FFC747` солнечный, `#5CCC99` мятный, `#66AEF2` небесный,
  `#A88CED` сиреневый; фон-крем `#FFFAED` → `#FFEBEF`; текст-чернила `#332941`. Скруглённый
  дружелюбный шрифт (web-аналог SF Rounded — напр. **Nunito**, есть кириллица).
- **Копирайтинг:** JTBD из SPEC §11 — «снять тревогу, что делаю прикорм неправильно»;
  «Четыре опоры» из README; privacy-стойкость как фича. Маскот Pudding — эмоция и лицо.
- **Dependencies:** только статика; деплой Vercel/Netlify (free). Никаких сторонних
  скриптов/трекеров.

## Development Approach

- Подход: чистый фронтенд-контент. **Юнит-тестов нет** (статический HTML/CSS) — это явно
  отмечено в задачах. Валидация — превью в браузере, проверка адаптива (моб/деск),
  link-check (нет битых ссылок), HTML/контраст-проверка.
- Двуязычность: дублирование страниц (RU в корне, EN в `/en/`) + переключатель языка в
  шапке. Простейший надёжный путь для статики, без i18n-рантайма.
- **Плейсхолдеры (🙋 USER):** домен, реальные скриншоты/видео, ссылка на App Store
  (`appStoreURL`), support-email.

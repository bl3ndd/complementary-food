# E2E-кейсы (XCUITest)

Полная карта маршрутов/кнопок приложения. Автоматизированные кейсы живут в `UITests/`
(таргет `PrikormUITests`, схема Prikorm → Test). Прогон — на устройстве (Cmd-U),
симуляторы не используем.

**Изоляция:** тесты запускают апп с аргументом `-uitest` → SwiftData-стор **in-memory**
(реальные данные на устройстве не затрагиваются), флаг дисклеймера чистится на старте,
язык принудительно RU (`-AppleLanguages (ru)`). Состояние задаётся сидом
`-uitest-seed=<name>` (см. `Sources/App/UITestSupport.swift`):

| Сид | Состояние |
|---|---|
| *(без сида)* | чистая установка → онбординг |
| `child` | ребёнок «Ника», пустой дневник |
| `introducing` | + брокколи «вводится», день 1 |
| `window-done` | + брокколи «вводится», окно (3 дн) прошло |
| `introduced` | + брокколи «введён» + кормление сегодня |
| `rich` | введённые брокколи/желток (желток «пора освежить»), кабачок вводится, треска на паузе, планы (яблоко сегодня, груша завтра), кормления с заметкой и реакцией |

## 1. Онбординг + дисклеймер-гейт — `OnboardingUITests`
- **E-ONB-01** Полный проход: Welcome → «Далее» → Ребёнок (имя вводится) → План → «Что уже ввели» → «Погнали! 🚀» → гейт «Прежде чем начать» → «Понятно» → таббар. ✅ авто
- **E-ONB-02** «Назад» возвращает на предыдущий шаг, введённое имя не теряется. ✅ авто
- **E-ONB-03** Имя можно пропустить (кнопка активна всегда). ✅ авто (implied)
- **E-ONB-04** Отмеченный «уже введён» продукт: в коллекции/каталоге «Введён», в дневнике записи НЕТ. ✅ авто
- **E-ONB-05** Параметры плана из онбординга доезжают до профиля. ⚪ юнит (FeedingProfileTests)

## 2. Главная — `DashboardUITests` (сид rich)
- **E-DASH-01** Hero: приветствие, имя, возраст, капсула «записей: N». ✅ авто
- **E-DASH-02** Плитка «Записать» → выбор продукта (поиск) → лист кормления → «Сохранить» → запись в «Дневник за сегодня». ✅ авто
- **E-DASH-03** Плитка «Реакция» → лист реакции → сохранение с бейджем реакции. ✅ авто (в FoodFlow)
- **E-DASH-04** «Отмена» в листе записи возвращает к списку продуктов, не на главную. ✅ авто
- **E-DASH-05** «Отмена» в списке продуктов → главная. ✅ авто (implied)
- **E-DASH-06** Тап записи «Дневник за сегодня» → правка («Запись»). ✅ авто
- **E-DASH-07** «Запланировать» → лист планирования. ✅ авто
- **E-DASH-08** «Сейчас вводишь» строка («День N из M») → карточка продукта. ✅ авто
- **E-DASH-09** Коллекция: тап иконки → карточка. ⚪ покрыто E-DASH-08/каталогом
- **E-DASH-10** «Вся коллекция» → таб Каталог. ✅ авто
- **E-DASH-11** Карточка «Аллергены» (+бейдж due на табе) → таб Аллергены. ✅ авто

## 3. Каталог — `CatalogUITests`
- **E-CAT-01** Секции по категориям; строка → карточка. ✅ авто (через поиск)
- **E-CAT-02** Поиск: вхождение, опечатка («брокол»), по категории («овощи»). ✅ авто
- **E-CAT-03** Бейдж статуса в строке (Введён/Вводится). ✅ авто
- **E-CAT-04** Свой продукт: «Добавить свой продукт» → имя → «Добавить» → секция «Свои продукты», сразу «Введён». ✅ авто
- **E-CAT-05** Удаление своего: свайп → «Удалить» → алерт → удалён. ✅ авто

## 4. Карточка продукта (стейт-машина) — `FoodStateMachineUITests`
- **E-FOOD-01** «Не введён»: дата старта + «Начать введение» → «Вводится», кольцо, «День 1 из 3». ✅ авто (сид child)
- **E-FOOD-02** «Вводится»: primary «Записать кормление»; меню «…»: «Была реакция», «Приостановить ввод». ✅ авто
- **E-FOOD-03** Окно прошло: primary «Ввёл успешно ✅» → поздравление → «Введён» + сводка «Кормлений: N». ✅ авто (сид window-done)
- **E-FOOD-04** «Введён»: primary «Записать кормление»; «…»: «Была реакция», «Пометить аллергию». ✅ авто
- **E-FOOD-05** «Пометить аллергию» → алерт → «Аллергия» (предупреждение) → «Вернуть в оборот (врач разрешил)» → «Вводится». ✅ авто (сид introduced)
- **E-FOOD-06** Пауза: «…» → «Приостановить» → алерт → «Пауза» → «…» «Попробовать через 2 месяца» («Напомним…») → «Возобновить ввод» → «Вводится». ✅ авто
- **E-FOOD-07** Повторный ввод НЕ скипает окно: после resume — «День 1 из 3», primary «Записать кормление» (не «Ввёл успешно»). ✅ авто (сид window-done)
- **E-FOOD-08** «Чем полезен» — аккордеон раскрывается (нутриенты-чипы). ✅ авто
- **E-FOOD-09** История: тап строки → правка; «Удалить запись» с алертом. ✅ авто
- **E-FOOD-10** Бэкдейт кормления двигает окно назад. ⚪ юнит (FeedingServiceTests)

- **E-PRF-04** Язык: пикер есть. ✅ авто (наличие); смена+перезапуск — 🔶 ручной (меняет реальные настройки)
